# Track-quality metrics

Living documentation of what `coolScore` measures, why, and how it's
validated. If you change the weights or add a metric, update this file in
the same commit.

## Three gates + a weighted sum

`coolScore(track)` returns 0 unless **all three** of these hold:

1. **Survival** — `terminus.reason === "endOfSpec"`. The rider must not
   eject, the sled must not break, the ride must not stall. A track no
   one rides is not a cool track.
2. **Contact floor** — `contactFractionLive ≥ 0.15`. The rider must
   actually be ON the track for at least 15% of its live frames.
   Otherwise it fell off the end of the geometry and is in free-fall —
   which trivially maximizes vertical trajectory range and would fool
   any size-based metric.
3. **Slow-slide ceiling** — `slowSlideFraction ≤ 0.5`. At most half of
   the contact frames may have `|vx| < 1.5 px/f`. A rider stuck oscillating
   in a local minimum (bouncing in place inside a halfpipe basin) has
   `slowSlideFraction` near 1.0. The oscillations can fake variety via
   bounce events and vy flips, but the rider isn't actually riding. The
   canonical failure: the original `compose_aerial` output, which had
   `meanVxSliding=0.35` after frame ~400 and scored a fake `cool=1938`
   before this gate was added.

If all three gates pass, `coolScore` is a weighted sum over the 8 features
below.

## The metrics

| Metric | Family | What it measures | Default weight | Why this weight |
|---|---|---|---|---|
| `angleStdDeg` | geometric | Length-weighted stddev of segment angles | 30 | Primary discriminator. Bland tracks (single curve) have ≈ 5°; varied tracks ≈ 25°. |
| `angleEntropyBits` | geometric | Shannon entropy of 12-bin angle histogram | 200 | Primary discriminator. Bland ≈ 0.1 bits, varied ≈ 1.9 bits. |
| `verticalExtentPx` | geometric | `max(y) - min(y)` of line endpoints | 0.2 | Size-based — light weight to avoid rewarding "big" over "varied". |
| `spreadEfficiency` | geometric | `boundingArea / totalLineLength` — how much screen the geometry covers per line | 0.1 | Size-based — light. |
| `eventRatePerSec` | behavioral | (landings + bounces + kicks) / live seconds | 30 | Moderate — high rate ≠ cool by itself (a flat slide with bouncy stubs can fake it). |
| `eventTypeEntropyBits` | behavioral | Shannon entropy over {landing, bounce, kick} mix | 100 | Discriminator. Mixed-event tracks (kicks AND bounces AND landings) are more interesting than landings-only. |
| `trajectoryVerticalPx` | behavioral | `max(y) - min(y)` of rider position over **contact frames only** | 0.1 | Size-based — light. Contact-only because free-fall would fake this. |
| `vySignFlips` | behavioral | Count of vy sign changes during **contact** | 50 | Primary discriminator. A rider that goes up-and-down repeatedly is doing something visually interesting. |

## Music-specific metrics (not in coolScore)

These ride alongside but are reported separately. A track can be cool
without any music sync — sync is a different axis.

| Metric | Meaning |
|---|---|
| `eventCoveragePct` | detected events ÷ beat count × 100 |
| `onBeatAdherencePct` | fraction of beats with at least one detector event within ±2 frames |
| `meanBeatOffsetFrames` | mean of nearest-event offsets per beat |

## Validation

The `tests/metrics_validation.test.ts` suite enforces:

1. **Separation gate** — `min(coolScore over cool refs)` > `max(coolScore over bland refs)` with margin > 20%.
2. **Pairwise** — every individual cool ref scores higher than every individual bland ref.
3. **Per-metric directionality** — `mean(cool) > mean(bland)` for each of the four primary discriminators (`angleStdDeg`, `angleEntropyBits`, `vySignFlips`, `eventTypeEntropyBits`).
4. **Robustness** — synthetic_cool (a robust authored anchor) median across ±0.3 px jitter stays above all bland medians.
5. **Determinism** — same track scored twice returns identical cool.
6. **Hard gates** — a non-surviving track returns 0; a contact-floor failure returns 0.

A translation-invariance test was attempted and dropped: lr-core's
floating-point trajectory is chaotic, and translating a fragile track
produces a different simulation. The metric uses only deltas (atan2 of
dx,dy; maxY−minY) so it leaks no absolute-position info; the determinism
test is the actual robustness guarantee we care about.

## How to re-tune weights

1. Edit `DEFAULT_COOL_WEIGHTS` in `scripts/lib/metrics.ts`.
2. Run `npx vitest run --no-file-parallelism tests/metrics_validation.test.ts`.
3. If the separation gate fails, weights are wrong — adjust until it passes.
4. If you can't make any weight assignment pass, the *family* of features
   is insufficient — add a new metric or replace an existing one.
5. Run `npx tsx scripts/dump_reference_metrics.ts` to see the full per-metric breakdown.
6. Update this file with the new weights and rationale.
